# Local Connection Server & ESP32 Arduino Integration Guide

This document describes the offline/local connection status mechanism implemented between the Flutter mobile application and the ESP32 ECG device.

---

## 📐 Architecture Overview

Instead of relying on a cloud-based Socket.io server to verify whether the ECG device has successfully joined the phone's Hotspot (which suffers from latency and cold starts), the connection status is sent directly over the local network.

```
+--------------------------+                  +-------------------------+
|                          |  (Hotspot Wi-Fi) |                         |
|   ESP32 ECG Device       | =================>|   Flutter Mobile App    |
|   (Client)               |                  |   (Local HTTP Server)   |
|   IP: 192.168.43.XX      |                  |   IP: 192.168.43.1      |
|                          |                  |                         |
+--------------------------+                  +-------------------------+
       |                                                   |
       |--- [HTTP GET: status (true/false)] -------------->|  Runs on Port: 9000
       |--- [HTTP GET: periodic heartbeat] ---------------->|  Updates UI dynamically
```

---

## 🌐 Local Server Endpoints (Flutter App)

The mobile application starts a lightweight HTTP server on port **9000** binding to `anyIPv4`. It exposes the following local endpoints:

### 1. Device Status
* **Endpoint**: `/device-status`
* **Method**: `GET`
* **Query Parameters**:
  * `connected` (bool): `true` if connected, `false` if disconnected.
  * `deviceId` (string): Unique identifier of the device (e.g., `ESP_ECG_123`).
* **Description**: Called by the ESP32 on initial connection or transition to notify the app of its state.
* **Response**: `{"status": "ok"}`

### 2. Heartbeat
* **Endpoint**: `/heartbeat`
* **Method**: `GET`
* **Query Parameters**:
  * `deviceId` (string): Unique identifier of the device.
* **Description**: Called by the ESP32 every 5 seconds to announce it is still alive. If the app does not receive this request for **12 seconds**, it marks the device as disconnected.
* **Response**: `{"status": "ok"}`

## 🔌 ESP32 Client Configuration (Arduino Side)

When the ESP32 joins the phone's hotspot, the gateway IP of the network is the phone itself. The ESP32 resolves this gateway dynamically using:
```cpp
IPAddress gateway = WiFi.gatewayIP();
```
It then communicates with `http://<gateway_ip>:9000`.

### Connection Transition Hook
The ESP32 monitors its connection status transitions inside the main `loop()` function. If the Wi-Fi status switches from disconnected to connected, it fires:
`GET http://<gateway_ip>:9000/device-status?connected=true&deviceId=ESP_ECG_123`

### Heartbeat Loop
Every 5 seconds, if the Wi-Fi connection is stable, it sends:
`GET http://<gateway_ip>:9000/heartbeat?deviceId=ESP_ECG_123`

## ⚙️ A-Z Step-by-Step setup

1. **Upload ESP32 Firmware**:
   Open Arduino IDE and upload the **ESP32 ECG Firmware v8.2** code onto your ESP32 board.
   
2. **Start the Mobile Hotspot**:
   Ensure your phone's Personal Hotspot is turned **ON**.
   * ⚠️ **Crucial**: Select **2.4 GHz Band** (Android) or enable **Maximize Compatibility** (iPhone). The ESP32 cannot connect to a 5 GHz band.

3. **Open the Mobile App**:
   On launch, the Flutter app starts the local server on port `9000` automatically.
   
4. **Provision the Wi-Fi**:
   * Navigate to the connection screen.
   * Connect your phone's Wi-Fi to the `ECG_Setup` network generated by the ESP32.
   * Input your hotspot SSID and Password.
   * Tap **Configure ECG**.
   * Turn **OFF** your phone's Wi-Fi and turn **ON** your Personal Hotspot.
   * The ESP32 will automatically connect to your hotspot and notify the local server. The UI will instantly display "Device Connected successfully!".
